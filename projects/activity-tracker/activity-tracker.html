Task Brief

This coursework was focused on building a complete Android application, demonstrating the knowledge we gained throughout the course

The brief expressly requested a 'basic running tracker' with features including

  • Logging the movement of the user when they go running
  • Saving the movement data
  • Allowing the user to inspect their data in an appropriate manner
  • Allowing the user to annotate their data in a useful manner
  • In principle, allowing other applications to make use of the data

Features

Movement tracking with Android's Location Manager

Live statistics whilst recording an exercise

  • Elapsed time
  • Distance
  • Maximum speed/elevation
  • Average speed

User friendly permissions requesting

Activity tagging

  • Activities can be tagged once an exercise is recorded
  • Tags can be changed when later viewing an exercise
  • Tags can be easily added from the tag selection section and are automatically removed if not in use

Filtering by tag in exercise list

Statistics calculated and made available when viewing an exercise

GPX format used to store route information which can be exported to be used elsewhere

Route viewable and drawn through Google Maps widget

Post Analysis

The result of this coursework was a well-functioning activity tracker that was provided a pretty standard interface and had a number of useful features

This application could definitely be improved upon, its shortcomings mostly the result of time constraints. There are many features that would greatly improve the final result which I may look to implement should I have spare time in the future. A greater consistency with Material design principles would definitely improve the familiarity of the interface.
