Voxel Meshing

Dissertation project focused on rendering

Description

For my dissertation project I chose to attempt implementing an efficient voxel renderer from the ground up. I had recently picked up some lower-level c++ side projects and had become interested in playing with opengl

With gameplay potential in an infinitely customisable world (see Minecraft) and greater applications in medicine or teaching, voxel rendering was a great dissertation topic.

I knew that this could get out of hand quickly so I focused primarily on some rendering techniques that could optimise a voxel environment. This had obviously been done before but the exploration is always a good thing.

  • Greedy meshing
  • Instance rendering
  • Chunking


Source code isn't available for this project

Post-analysis

This project was a very mixed bag in terms of results for me. Ultimately I achieved the goals outlined at the start of the project, exploring efficient methods for voxel rendering. However I had hoped for much more, creating a solid base for a voxel engine that I could take further. This code however sits standalone as there simply wasn't enough time to create this is an extensible way

I also hoped to explore more rendering methods, especially octrees as it scales up the potential massively. The code base is also messy and I am not happy sharing it publicly as a result. To do so it would require a lot of clean up

The context in which this project was undertaken isn't quite what I hoped for the outcome. Whilst I learnt plenty of useful information, I had wanted to create a game engine which just isnt feasible for an undergraduate dissertation

Using the knowledge gained however I do want to get onto an open source voxel engine implementing some fun gameplay ideas and some complex methods. In this way I can hopefully maximise the value gained from this experience
