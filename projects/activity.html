Activity Tracker

Activity logging on Android using multiple sensors

Description

This project was the final coursework for my final year mobile programming course, aimed at bringing together all we had learned into a single application

This course had covered topics like Android activities, permissions, lifecycle, storage, and more

The brief requested a 'basic running tracker' with a list of features including

  • Logging the movement of the user when they go running
  • Saving the movement data
  • Allowing the user to inspect their data in an appropriate manner
  • Allowing the user to annotate their data in a useful manner
  • In principle, allowing other applications to make use of the data

Post-analysis

This application boasted a number of features such as: gps tracking with Android's location manager, live statistics whilst recording an exercise (elapsed time, distance, maximum speed/elevation, average speed), permission requests, activity tagging, tag filtering, gpx format save, visualised route on google maps widget

Most of the requested features were implemented and a solid mark was achieved

As always there are things that could have been improved with this application; most of the application's shortcomings were a result of a lack of time. Future attempts that are less time constrained would improve the user experience by following Material UI design guidelines to provide a more familiar feel. Additionally, more details about activities and more types of activities should be added
