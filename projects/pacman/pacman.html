Task Brief

We were tasked with refactoring a pacman project created in a previous year

One thing in particular I wished to change was the movement system

  • It was using a poorly done and confusing floating system
  • Pacman and ghosts moved linearly in undefined spaces between walls

I wished to implement a vector based movement system

  • The level would be represented as a grid array with clear paths for entities to move in
  • Entities would move between grid squares fluidly without floating in-between

Vector2

I created a vector class that would define (x, y) coordinate pairs and make my life easier
Click here for the code!

This class implemented a variety of behaviours

  • Stored 2 public doubles that would serve (in our case) as x and y coordinates
  • Provided some static Vector2s that represented cardinal directions (very helpful in better relaying what code does in later code
  • Allowed for comparison of two Vector2s as well as standard operations (+, -, *, /) either by another Vector2 or by a constant value
  • Linear interpolation between two Vector2s
  • Straight line distance between two Vector2s

MoveableEntity
